# 에러 해결 기록

> 개발 중 마주친 에러와 해결 과정을 날짜별로 기록

---

## 2026-03-13 | eBPF 첫 빌드 에러 3종

### 배경
`agent/tcp_trace.bpf.c` 를 처음 컴파일할 때 발생한 에러들.
`clang -target bpf` 로 BPF 바이트코드를 생성하는 과정에서 헤더 파일 문제가 연속으로 발생.

---

### 에러 1: `asm/types.h` file not found

**발생 상황**
```
In file included from tcp_trace.bpf.c:12:
In file included from /usr/include/linux/bpf.h:11:
/usr/include/linux/types.h:5:10: fatal error: 'asm/types.h' file not found
```

**원인**

`-target bpf` 로 컴파일할 때 clang은 BPF 가상머신 타겟으로 빌드함.
이때 `linux/bpf.h` → `linux/types.h` → `asm/types.h` 순서로 헤더를 찾는데,
`asm/types.h` 는 CPU 아키텍처별로 다른 경로에 있음:

```
/usr/include/
├── asm/              ← 심볼릭 링크 없음 (BPF 타겟은 여기를 못 찾음)
├── x86_64-linux-gnu/
│   └── asm/
│       └── types.h   ← 실제 파일은 여기 있음
```

BPF 타겟 컴파일 시 기본 include 경로에 `x86_64-linux-gnu/` 가 빠져있어서 발생.

**해결**
Makefile의 `BPF_CFLAGS`에 해당 경로를 명시적으로 추가:
```makefile
BPF_CFLAGS := -g -O2 -target bpf -D__TARGET_ARCH_x86 \
              -I/usr/include/x86_64-linux-gnu \   # ← 추가
              -I/usr/include
```

---

### 에러 2: `struct pt_regs` incomplete definition

**발생 상황**
```
tcp_trace.bpf.c:76:5: error: incomplete definition of type 'struct pt_regs'
int BPF_KPROBE(handle_tcp_connect, struct sock *sk)
```

**원인**

`BPF_KPROBE` 매크로는 내부적으로 `PT_REGS_PARM1(ctx)` 를 사용하는데,
이게 `struct pt_regs` (CPU 레지스터 상태를 담는 구조체)를 필요로 함.

`bpf_tracing.h` 는 `__TARGET_ARCH_x86` 매크로가 정의되면
`PT_REGS_PARM1` 을 x86용으로 정의하지만,
실제 `struct pt_regs` 선언은 `asm/ptrace.h` 에 있고
이 파일이 자동으로 include 되지 않음.

```
BPF_KPROBE 매크로
    └── PT_REGS_PARM1(ctx)
            └── ctx->rdi  (x86_64 첫번째 인자 레지스터)
                    └── struct pt_regs 필요! ← 이게 없어서 에러
```

**해결**
`tcp_trace.bpf.c` 상단에 직접 include 추가:
```c
#include <asm/ptrace.h>   // ← 추가: struct pt_regs 정의
```

### 에러 3: `bpf_ntohs` undeclared

**발생 상황**
```
tcp_trace.bpf.c:109:16: error: call to undeclared function 'bpf_ntohs'
e->dport = bpf_ntohs(BPF_CORE_READ(sk, __sk_common.skc_dport));
```

**원인**

`bpf_ntohs()` 는 네트워크 바이트 오더(big-endian)를 호스트 바이트 오더(little-endian)로
변환하는 BPF 전용 함수.

일반 C의 `ntohs()` 와 달리 BPF 전용 버전이 따로 있으며,
`bpf/bpf_endian.h` 헤더에 선언되어 있는데 이 파일을 include 하지 않았음.

참고로 왜 변환이 필요한가:
```
네트워크 프로토콜은 big-endian (상위 바이트 먼저)
Intel x86_64 는 little-endian (하위 바이트 먼저)

포트 번호 443 의 메모리 표현:
  big-endian (네트워크):    0x01 0xBB
  little-endian (x86):     0xBB 0x01

bpf_ntohs() 없이 읽으면 포트가 뒤집혀서 잘못된 값이 나옴
```

**해결**
```c
#include <bpf/bpf_endian.h>   // ← 추가: bpf_ntohs(), bpf_htons() 등
```

## 최종 해결된 include 목록 (tcp_trace.bpf.c)

```c
#include <linux/bpf.h>
#include <asm/ptrace.h>         // struct pt_regs (BPF_KPROBE 매크로용)
#include <bpf/bpf_helpers.h>    // bpf_get_current_pid_tgid() 등
#include <bpf/bpf_tracing.h>    // BPF_KPROBE 매크로
#include <bpf/bpf_core_read.h>  // BPF_CORE_READ (CO-RE용)
#include <bpf/bpf_endian.h>     // bpf_ntohs()
```

## 최종 Makefile BPF_CFLAGS

```makefile
BPF_CFLAGS := -g -O2 -target bpf -D__TARGET_ARCH_x86 \
              -I/usr/include/x86_64-linux-gnu \
              -I/usr/include
```
