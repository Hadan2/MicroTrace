# NetSim — 연관 엣지 & 데이터 소유권 (Edges & Ownership)

> 역할: **엣지맵(EdgeMap)**. "기능들이 어떻게 엮이고(엣지), 무엇이 외부 소유라 손대면 안 되는가(경계)"를 적는다.
> **코드 수정 전에 이 문서의 해당 행을 먼저 확인**한다.
>
> ⚠️ **현재 NetSim은 기획 단계 — 코드 미구현.** 아래는 채울 골격이다. 코드가 생기면
> `netsim.code.md`(코드맵)를 먼저 만들고, 여기의 각 표를 실제 소스 근거로 채운다.
> 개념·기획은 [`../guide/overview.md`](../guide/overview.md), 전문은 `../reference/{netsim,integration}.md`.

---

## A. 데이터 소유권 표 — "이건 외부 소유, 수정 금지" ★최우선 확인★

> NetSim이 외부 소유로 다룰 후보(구현 시 실제 근거로 확정):
> - Docker API가 준 타겟 컨테이너 **PID/veth** (읽기 전용, 우리가 만든 값 아님)
> - `tc/netem` 커널 규칙 상태 (커널 소유 — 우리는 적용/해제만)

| 데이터 | 소유자 | 가변성 | 코드 근거 | 규칙 |
|---|---|---|---|---|
| {{타겟 PID/veth}} | Docker API | read-only | {{구현 시}} | 원시값 수정 금지, 조회만 |

---

## B. 연관 엣지 표 — "X를 건드리면 Y를 봐라"

> ★ 핵심 크로스-프로젝트 엣지(통합 모드): NetSim이 장애를 **주입(원인)**, MicroTrace가 **감지·분석(결과)**.
> MicroTrace 토폴로지 클릭 → NetSim 제어판 호출의 인터페이스(REST API 포맷)가 두 프로젝트를 결합한다.

| 건드리는 기능 | 같이 봐야 할 기능 | 엣지 종류 | 왜 |
|---|---|---|---|
| {{NetSim REST API 포맷}} | MicroTrace 통합 모드 호출부 | 포맷 공유(크로스-프로젝트) | 통합 모드에서 MicroTrace가 이 API를 호출 (integration.md) |

---

## C. 의미론적 충돌 주의 지점 (빌드는 통과하지만 깨지는 곳)

1. **tc 규칙 롤백** — 종료 시 signal handler로 자동 롤백. 누락하면 좀비 규칙이 남아 다음 실험을 오염(빌드는 통과).
2. {{구현 시 추가}}

---

## D. 버그 점검 시 체크 순서

1. 타겟 식별(PID/veth)이 맞나 — Docker API 조회 결과 확인.
2. tc 규칙이 실제 적용/해제됐나 — 커널 상태를 로그로 찍어 확인(추측 금지).
3. {{구현 시 추가}}
