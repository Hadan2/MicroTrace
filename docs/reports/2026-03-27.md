# 03.27 트러블슈팅

---

## HTTP Keep-Alive 연결 재사용 안 됨

### 증상
- `DisableKeepAlives: false` (Keep-Alive 활성화) 로 설정했는데
- service_a → service_b 요청마다 MicroTrace에 `[CONNECT]` 가 계속 찍힘
- service_b 로그에서 포트 번호가 매 요청마다 바뀜

```
[B] GET /ping from 127.0.0.1:48364
[B] GET /ping from 127.0.0.1:48370  ← 포트가 다름 = 새 TCP 연결
[B] GET /ping from 127.0.0.1:48372
```

---

### 원인 1: resp.Body를 읽지 않고 바로 Close()

Go의 HTTP 클라이언트는 **Body를 끝까지 읽고 닫아야** 연결을 커넥션 풀에 반환함.

```go
// ❌ 잘못된 코드 - Body를 읽지 않고 닫음 → 연결 재사용 불가
resp.Body.Close()

// ✅ 올바른 코드 - Body를 끝까지 읽고 닫음 → 연결 풀에 반환됨
io.Copy(io.Discard, resp.Body)
resp.Body.Close()
```

**왜 그럴까?**
```
HTTP 응답 Body = 네트워크 스트림
                        ↓
Body를 다 읽어야 → "이 응답 다 받았다" 확인
                        ↓
연결을 안전하게 풀에 반환 가능
                        ↓
Body를 안 읽고 닫으면 → 이 연결은 오염된 상태로 판단 → 재사용 불가 → 새 연결
```

---

### 원인 2: service_b에 IdleTimeout 미설정

Go `http.Server`는 `IdleTimeout`이 0이면 `ReadTimeout` 값을 사용함.
`ReadTimeout: 5초`로만 설정했더니 Keep-Alive 연결이 5초 후 끊어짐.

```go
// ❌ 잘못된 코드
srv := &http.Server{
    Addr:        ":8080",
    ReadTimeout: 5 * time.Second,
    // IdleTimeout 없음 → ReadTimeout(5초) 사용 → 연결 자주 끊김
}

// ✅ 올바른 코드
srv := &http.Server{
    Addr:        ":8080",
    ReadTimeout: 5 * time.Second,
    IdleTimeout: 60 * time.Second, // Keep-Alive 연결 60초 유지
}
```

---

### 해결 후 확인

`ss -tn | grep 8080` 으로 연결 상태 확인:
```
ESTAB  127.0.0.1:41584  127.0.0.1:8080  ← 포트 고정 = 연결 재사용 중
```

MicroTrace 출력:
```
[CONNECT] PID: 11932  COMM: service_a  ->  127.0.0.1:8080  latency: 36 us
← 이후 요청에서는 CONNECT 없음 (Keep-Alive로 재사용)
```

---

### 교훈

1. **Go HTTP 클라이언트에서 항상 Body를 끝까지 읽고 닫을 것**
   ```go
   io.Copy(io.Discard, resp.Body)
   resp.Body.Close()
   ```

2. **Go http.Server 설정 시 IdleTimeout을 명시할 것**
   ```go
   IdleTimeout: 60 * time.Second
   ```

3. **Keep-Alive 동작 확인 방법**
   ```bash
   ss -tn | grep <포트>
   # 포트 번호가 고정 → 연결 재사용 중
   # 포트 번호가 변경 → 매번 새 연결
   ```
